# IT用語クイズ アプリ

勉強会2026向けIT用語クイズアプリです。  
出題者PC・モニター・参加者スマホをリアルタイムで同期します。

---

## 画面構成

| 画面 | URL | 用途 |
|------|-----|------|
| ホーム | `/` | 各画面へのリンク・接続先QR表示 |
| 出題者画面 | `/host` | 問題操作・○×判定・早押しリセット・得点管理 |
| モニター画面 | `/monitor` | プロジェクター・大画面向け（操作なし） |
| 早押しボタン | `/buzzer` | スマホ・タブレットから参加 |

---

## 起動方法

### 1. 初回のみ：パッケージインストール

```bash
cd IT_Quiz
npm install
```

### 2. 開発サーバー起動（推奨）

```bash
npm run dev
```

- **サーバー（WebSocket）**: `http://localhost:3001`
- **フロントエンド（Vite）**: `http://localhost:5173`

ブラウザで `http://localhost:5173` を開く。

### 3. 本番起動（ビルドして1ポートで動かす場合）

```bash
npm run build
npm run start
```

- サーバーが `http://localhost:3001` でフロントエンドも含めて起動

## スマホから接続する方法

1. PCとスマホを **同じWi-Fi** に接続する
2. `npm run dev` または `npm run start` でサーバーを起動
3. サーバー起動時にターミナルに表示されるURLを確認：

```
📱 ネットワーク: http://192.168.x.x:3001
🔔 早押しボタン: http://192.168.x.x:3001/buzzer
```

4. スマホのブラウザで上記の早押しボタンのURLを開く
5. または、ホーム画面（PC側）に表示されるURLをスマホでアクセス

> **PCのIPアドレスを確認したい場合**  
> コマンドプロンプトで `ipconfig` → 「IPv4アドレス」の値

## 進行手順（当日の流れ）

```
1. 司会者のPCで npm run dev （または npm run start）を起動
2. 司会者はブラウザで /host を開く
3. プロジェクター用PCで /monitor を開く
4. 参加者はスマホで /buzzer を開き、名前を入力して「参加する」
5. クイズスタート！
```

### 司会者の操作方法

| 操作 | 説明 |
|------|------|
| 答えを表示する | 問題の解説を全画面に表示 |
| 次の問題 / 前の問題 | 問題を切り替え（早押しも自動リセット） |
| ◯ 正解 | 回答権を持つ人に+1点 |
| × 不正解 | 回答権を次の人に移す |
| ↩ 戻す | 回答権を前の人に戻す |
| リセット | 早押しキューをクリア（同じ問題でやり直し） |

### 早押しの仕組み

1. 参加者がスマホで早押しボタンを押す
2. 押した順番が全画面に表示される（1位〜）
3. 1位の人が「回答権」を持つ
4. 司会者が答えを聞いて ◯ または × を押す
   - ◯ → その人に得点
   - × → 次の順位の人に回答権が移る
5. 次の問題に進む前に「リセット」を押す

## 問題データの編集

`public/questions.json` を編集することで問題を追加・変更できます。

```json
{
  "id": 68,
  "category": "カテゴリ名",
  "term": "用語名",
  "explanation": "説明文",
  "contributor": "出題者の名前"
}
```

サーバーを再起動すると反映されます。

## トラブルシューティング

| 問題 | 対処 |
|------|------|
| スマホから繋がらない | PCとスマホが同じWi-Fiか確認。PCのファイアウォールでポート3001を許可 |
| `npm run dev` でエラー | `npm install` を再実行 |
| 画面が更新されない | ブラウザをリロード（F5）。2秒後に自動再接続します |
| 得点がリセットされた | ページをリロードしても名前が同じなら得点は保持されます |
